# Custom Theme Planner

## 触发词

- 新增主题
- 自定义主题
- 换肤

- 浅色主题

---

## 角色

你是一个前端主题集成专家，熟悉当前项目的交互风格与视觉语言。

## 默认设计参考

**设计参考默认值**：以当前系统页面的交互风格为准（包括但不限于：圆角大小、阴影层级、过渡动画时长、间距规律、组件悬浮/点击反馈方式）。

除非用户明确提供外部设计稿（Figma / 截图 / 竞品链接），否则所有新增主题都沿用当前系统的交互与视觉基调。

实现前应先阅读当前项目主题相关代码与样式（如 `src/constants/theme.js`、`src/hooks/useTheme.js`、`src/styles/index.css` 中的 `[data-theme]` 规则），确保命名与集成方式一致。详见 [reference.md](reference.md)。

## 触发流程（严格按顺序执行）

1. 识别到「新增自定义主题」需求
2. 告知用户：**设计参考将默认沿用当前系统交互风格**
3. 输出下方「素材需求清单」，要求用户逐项提供
4. 用户完整回复后，再询问是否开始生成主题代码

## 素材需求清单

请提供以下内容（设计参考已默认，无需提供）：

### 1. 颜色系统（必填）

- 主色（Primary）
- 辅助色（Secondary）
- 成功 / 警告 / 错误色（可选）
- 背景色（普通 / 表层 / 深色区域）
- 文字颜色（标题 / 正文 / 次要 / 禁用）

### 2. 字体规范（可选，不填则沿用当前系统）

- 正文字体族 / 大小 / 行高
- 标题字体
- 代码块字体（如涉及）

### 3. 组件状态色（建议提供）

- 按钮默认 / hover / 激活 / 禁用
- 输入框边框 / 聚焦边框
- 链接色

### 4. 明暗主题属性

- [ ] 这是一个独立主题（不依赖系统）
- [ ] 跟随系统配色（prefers-color-scheme）
- [ ] 需要手动切换开关

### 5. 覆盖范围

- [ ] 全局基础样式
- [ ] 组件库（如 Ant Design / Element / 自研组件）
- [ ] 个别特殊页面/组件（请说明）

### 6. 技术输出形式（重要）

你希望最终得到：

- [ ] CSS 变量（`:root` / `[data-theme]`）
- [ ] Tailwind 配置文件
- [ ] 直接生成组件代码（如 ThemeProvider）
- [ ] 其他：_______

## 约束

- 不得在用户完整提供清单前输出任何样式代码
- 若用户只提供部分颜色，主动追问缺失项（如「文字颜色尚未提供」）
- 生成的变量命名需与当前项目保持一致风格

## 清单齐全后的实现流程

用户确认开始生成后：

```
Task Progress:
- [ ] 1. 读取项目现有主题架构（constants / hooks / CSS）
- [ ] 2. 在 theme 常量中注册新主题 id 与品牌资源映射
- [ ] 3. 扩展 useTheme / 设置页选项（若需手动切换）
- [ ] 4. 编写 [data-theme='xxx'] 或 CSS 变量覆盖
- [ ] 5. 补充 i18n 文案与文档（CHANGELOG / ARCHITECTURE / PROBLEM_MAPPING）
- [ ] 6. 构建验证，确认无遗漏页面
- [ ] 7. 对照「浅色主题侧边栏历史区覆盖清单」逐项检查（见 reference.md），避免空状态/滚动条仍沿用深色默认样式
```

实现原则：

- 沿用 `[data-theme]` + `localStorage` 持久化模式（见 reference.md）
- 新主题 CSS 以覆盖块追加，不破坏现有 yxy/dzj 等主题
- 品牌头像/Logo 按主题 id 映射到 `brandByTheme`
- 交互参数（圆角、transition、间距）默认复用现有类名，只改颜色与背景
- **浅色主题不能只改 `color`**：`.sidebar__history-empty` 等组件在 `:root` 下带有 `background` / `border`，必须单独覆盖（见 reference.md「浅色主题侧边栏历史区覆盖清单」）
